**Hybrid Question System**

**Exam Question Source**  
- Subjects (often multiple)  
- Source (past papers, Century, Knowt, Web page)

**Questions/Task**  
- Course Unit (when should students be able to do this. Maybe based on textbook chapter. But that can change. Maybe use Subject Units which match to textbook chapters)  
- Source  
- Name  
- Description (rtf field that could hold screenshot of the question. Maybe we should record question and Answer here as well as a link if possible)  
- Link to knowt, YouTube answer  
- Answer rtf field  
- Question rtf field. These would make it possible for someone to do the question without going off the site.  
- Attached to a lesson plan?  
- Expected duration (mins, hours day)

**Subject Units**  
- List of topics and their order. Use to identify when a question can be assigned to a student

**Student Subjects**

**Question Attempts**  
- Date Completed  
- Result %  
- Due date (optional. It's possible that a student has already done the question before it is assigned. Need to decide if they must do the question again (so create a new record with new due date), accept a previous attempt (flag as incomplete or teacher suggested to do it again), or skip if already completed)  
- Completed  
- Teacher requested redo  
- Due Status (late, done, early)  
- Actual duration - how long the task took.

**Student Questions**  
- Difficulty Self Assessed (hard, OK, easy)  
- Difficulty Calculated (how hard the software thinks the question is for the student)  
- Status (Not Started, Completed)

**Teacher Help Status**  
- This might need to be a table where we can record the date the teacher helped  
- Total Attempts  
- Last score  
- Average score

**Parental Review**  
- Date  
- Questions included in the review

**Status**  
- Requested  
- Completed  
- Inquiry
